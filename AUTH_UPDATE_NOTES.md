# Auth.vue 登录/注册流程更新说明

## 更新时间
2026-04-18 21:30

## 主要改进

### 1. 统一的登录/注册页面
- **第一页永远是手机号输入页面**，不再有"登录"和"注册"的标签切换
- 系统自动识别用户身份（老用户/新用户）

### 2. 智能用户识别
当用户输入完整的10位手机号后：
- **老用户**：显示绿色提示 "✅ 老用户，点击按钮直接登录"
- **新用户**：显示红色提示 "🆕 新用户，请填写昵称完成注册"

### 3. 界面布局优化

#### 第一行：手机号输入
```
┌─────────────────────────┐
│ 手机号                   │
│ +1 | [请输入加拿大手机号] │
└─────────────────────────┘
```

#### 第二行：昵称输入（仅新用户可见）
```
┌─────────────────────────┐
│ 昵称                     │
│ [请使用群内昵称]         │
│ 💡 提示：昵称请使用群内昵称 │
└─────────────────────────┘
```

### 4. 动态提交按钮
- **新用户**：按钮文字为 "注册并登录"
- **老用户**：按钮文字为 "登录"
- 未满足条件时按钮禁用

### 5. 用户体验优化

#### 老用户流程
1. 输入手机号（10位）
2. 系统自动识别为老用户
3. 显示绿色提示
4. 点击"登录"按钮
5. 直接进入产品页面

#### 新用户流程
1. 输入手机号（10位）
2. 系统自动识别为新用户
3. 显示昵称输入框和红色提示
4. 填写群内昵称
5. 点击"注册并登录"按钮
6. 完成注册并进入产品页面

### 6. 技术实现

#### 数据存储
- 用户列表存储在 `localStorage.users`
- 当前用户存储在 `localStorage.currentUser`
- 每个用户包含：
  - `id`: 唯一标识符
  - `nickname`: 昵称
  - `phone`: 手机号（+1格式）
  - `createdAt`: 注册时间
  - `lastLoginTime`: 最后登录时间

#### 自动跳转
- 如果检测到已登录用户，自动跳转到产品页面
- 避免重复登录

### 7. 验证规则
- 手机号必须是10位数字
- 新用户的昵称不能为空
- 提示使用群内昵称

## 测试建议

### 测试场景1：新用户注册
1. 打开应用
2. 输入新的手机号（如：4161234567）
3. 确认显示昵称输入框
4. 填写昵称（如：张三）
5. 点击"注册并登录"
6. 验证是否跳转到产品页面

### 测试场景2：老用户登录
1. 打开应用
2. 输入已注册的手机号
3. 确认不显示昵称输入框
4. 确认显示绿色提示
5. 点击"登录"
6. 验证是否跳转到产品页面

### 测试场景3：已登录用户
1. 清除浏览器缓存
2. 重新打开应用
3. 验证是否自动跳转到产品页面
