# 前端硬编码问题整改方案

## 问题概述
在前端代码审查中发现多处硬编码问题，影响应用的可维护性、可扩展性和国际化能力。

## 硬编码问题清单

### 🔴 高优先级（安全和可维护性）

#### 1. API 配置硬编码 - `src/api.js`
- **`apikey: 'anon'`** - 硬编码的API密钥
- **`timeout: 10000`** - 硬编码的请求超时时间  
- **`baseURL: BASE_URL + '/api'`** - 硬编码的API路径前缀

#### 2. 敏感业务逻辑硬编码 - `src/pages/order/create.vue`
- **满减规则**: `subtotal.value >= 100 ? 10 : 0` (满100减10)
- **支付方式文本**: `"到店付款（自提时支付）"`

### 🟡 中优先级（国际化和可维护性）

#### 3. UI 文本硬编码 - 多个页面
- **商家信息标签**: `"商家名称:"`, `"地址:"`, `"Email:"`
- **默认提示文本**: `"暂无地址信息"`, `"暂无邮箱信息"`, `"暂无商家信息"`
- **页面标题**: `"订单商品"`, `"费用明细"`, `"提交订单"`
- **按钮文本**: `"加入购物车"`, `"立即购买"`, `"确认下单"` 等

#### 4. 路由路径硬编码 - 多个导航函数
- **页面路径**: `router.push('/goods')`, `router.push('/cart')`, `router.push('/user')`
- **商品详情路径**: `router.push(`/goods/${product.id}`)`

### 🟢 低优先级（样式一致性）

#### 5. 样式值硬编码 - 多个 Vue 组件
- **颜色值**: `#333`, `#666`, `#999`, `#ddd`, `#07c160`, `#06a852`
- **尺寸值**: `padding: 16px`, `font-size: 14px`, `border-radius: 8px` 等
- **媒体查询断点**: `(max-width: 375px)`

#### 6. localStorage 键名硬编码 - 多个页面
- **存储键名**: `'currentUser'`, `'cart'`

## 整改优先级和步骤

### 第一阶段：高优先级安全问题（立即处理）
1. **API密钥移至环境变量**
2. **API配置参数化**
3. **业务规则配置化**

### 第二阶段：中优先级可维护性（本周内）
1. **实现国际化(i18n)支持**
2. **路由命名规范化**
3. **常量提取到配置文件**

### 第三阶段：低优先级样式优化（后续迭代）
1. **创建CSS变量系统**
2. **设计令牌标准化**

## 技术实施方案

### 环境变量配置
```bash
# .env 文件
VITE_API_KEY=your_api_key_here
VITE_API_TIMEOUT=10000
VITE_API_BASE_PATH=/api
```

### 国际化配置
```javascript
// src/locales/zh-CN.js
export default {
  common: {
    merchantName: '商家名称',
    address: '地址',
    email: 'Email',
    noAddress: '暂无地址信息',
    noEmail: '暂无邮箱信息',
    noMerchant: '暂无商家信息'
  },
  order: {
    products: '订单商品',
    feeDetails: '费用明细',
    submitOrder: '提交订单',
    paymentMethod: '到店付款（自提时支付）'
  }
}
```

### 路由配置优化
```javascript
// src/router/index.js - 添加路由名称
{
  path: '/goods', 
  name: 'goods',
  component: () => import('../pages/goods/index.vue') 
}
```
